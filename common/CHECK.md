# Shared Check

Use this checklist before finalizing drafted, rewritten, reviewed,
explained, or generated work.

* The main point appears near the top.
* The output preserves the user's facts, intent, constraints, and
  technical claims.
* Claims are scoped to the evidence available.
* Facts, inferences, and recommendations are not blurred together.
* Uncertainty is stated directly when it affects the reader's
  decision.
* The structure matches the audience and task.
* The response avoids filler, inflated praise, empty apology, and
  decorative transitions.
* Private facts from examples or references were not copied into
  public guidance or generated text.
* Markdown prose uses `*` for unordered lists unless tool syntax
  requires otherwise.
* Verbatim legal text and externally specified tool syntax were left
  intact.
* The narrowest relevant task guidance was used when it exists.
